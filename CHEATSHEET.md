# Шпаргалка: Работа с ЕСН и масками

## 1. Создание масок на основании файла ЕСН

### Шаг 1.1: Построить индекс ЕСН
```bash
python cli.py ens build-index "data/_ЕНС_Крепеж_24.03.2026.xlsx" \
    -o models/ens_hardware.pkl \
    -c hardware
```

**Результат:**
- `models/ens_hardware.pkl` - TF-IDF индекс для поиска
- `models/ens_hardware.meta.json` - метаданные (количество записей, колонки)

### Шаг 1.2: Генерация масок для всех стандартов в ЕСН
```bash
python cli.py generate-masks \
    -d cache/masks.db \
    -i models/ens_hardware.pkl \
    --min-score 0.85
```

**Что происходит:**
- Для каждого (ГОСТ/ОСТ, тип) из ЕСН проверяет наличие маски
- Если маски нет — генерирует через LLM (Qwen3 → GPT-4 fallback)
- Валидирует на 10+ примерах из ЕСН
- Активирует если score ≥ 0.85

**Результат в БД:**
- Активные маски готовы к использованию
- Неактивные (draft) можно доработать

## 2. Обработка файла nomenclature1.xlsx для проверки валидности

### Вариант A: Быстрая проверка (только regex, без LLM)
```bash
python cli.py batch data/nomenclature1.xlsx \
    -d cache/masks.db \
    -i models/ens_hardware.pkl \
    -o results.json \
    --validate
```

**Проверяет:**
- ✅ Извлечение стандарта (Level 0)
- ✅ Наличие маски в БД (Level 1)
- ✅ Извлечение всех обязательных параметров (Level 6)
- ✅ Сопоставление с ЕСН (Level 6-7)

**Результат:**
- `results.json` с полями: `success`, `params`, `ens_code`, `confidence`
- Статистика: сколько разобрано автоматически, сколько требует внимания

### Вариант B: С генерацией недостающих масок (LLM)
```bash
python cli.py batch data/nomenclature1.xlsx \
    -d cache/masks.db \
    -i models/ens_hardware.pkl \
    -o results.json \
    --llm \
    --validate
```

**Дополнительно:**
- 🤖 Автогенерация масок для новых/неизвестных стандартов
- 🧪 Автовалидация сгенерированных масок
- 💾 Сохранение качественных масок (score ≥ 0.85) в БД

**Время выполнения:**
- 1000 записей: ~2-3 минуты (с LLM) vs ~10 сек (только regex)

## 3. Анализ перед полной обработкой

### Проверить покрытие файла индексом ЕСН
```bash
python cli.py ens analyze data/nomenclature1.xlsx \
    -i models/ens_hardware.pkl \
    -s 200
```

**Вывод:**
- Процент записей, разбираемых regex (ожидается 70-80%)
- Процент, требующий LLM (15-20%)
- Статистика по типам (болты, винты, шайбы)

## 4. Управление масками

### Просмотр статистики
```bash
python cli.py stats -d cache/masks.db
```

### Очистка низкокачественных масок
```bash
python cli.py cleanup -d cache/masks.db -t 0.5
```

### Поиск в индексе ЕСН (ручная проверка)
```bash
python cli.py ens search "Болт М12" -i models/ens_hardware.pkl -k 5
```

## Пороги качества (контрольные точки)

| Score | Статус | Действие |
|-------|--------|----------|
| ≥ 0.85 | ✅ Active | Используется в продакшене |
| 0.50-0.85 | ⚠️ Draft | Требует доработки или ручной проверки |
| < 0.50 | ❌ Rejected | Удаляется при cleanup |
