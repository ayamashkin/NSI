## Сводка проблем

### Критические (блокируют использование)

1. **Длина=null в Болт 31133-80** — `params` неполные, поиск в ЕНС отваливается
2. **Params пустые в Винт (4)-31503-80** — regex маска не срабатывает на текст
3. **Standard пустой у Шайба** — запятая в конце ломает StandardExtractor

### Средние (можно обойти)

4. **Покрытие переставлено** — `Окс.Фос.ЭФП` vs `Фос.Окс.ЭФП` — fuzzy matching не активируется
5. **Болт ГОСТ 7795-70** — маска некорректно парсит группу прочности и длину

### Низкие (косметические)

6. **ens_params=null** — ожидаемо, т.к. требует заполненного ens_code

## Рекомендации по исправлению

### 1. Исправить маску Болт 31133-80

Текущая маска:
```
^Болт(?:\s*\((?P<исполнение>\d+)\)\s*)?(?P<наружный_диаметр_диаметр_вписа>\d+)(?:-(?P<номинальный_диаметр_резьбы>\d+))?(?:-(?P<длина>\d+))?(?:-(?P<покрытие>[\w.]+))\s*-ОСТ\s*1\s*31133-80[\s,;.]*$
```

Проблема: `(?:-(?P<номинальный_диаметр_резьбы>\d+))?` захватывает `44` как `номинальный_диаметр_резьбы`, а `длина` остается null.

В тексте `Болт (2)-12-44-Окс...`:
- `12` → `наружный_диаметр_диаметр_вписа`
- `44` → `номинальный_диаметр_резьбы` (по маске)
- `длина` = null

**Но** в ручном сопоставлении `44` — это длина! Проблема в структуре маски (LLM сгенерировал неверно).

### 2. Исправить маску Винт 31503-80

Добавить `\s*` после `)`:
```
(?:\(\s*(?P<исполнение>\d+)\s*\)\s*-\s*)?
```

### 3. Очистка текста от пунктуации

Добавить в `standard_extractor.py`:
```python
text = text.rstrip(',.;: ')
```

### 4. Улучшить fuzzy matching

Активировать fuzzy matching для покрытия даже при score=1.0, если текстовые параметры не совпадают.
