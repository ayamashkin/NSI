## Скорректированный промпт для предотвращения ошибки

**Входные данные:**
Наименование номенклатуры — "{{NOMENCLATURE}}"

**Правила обработки:**
1. Определи тип изделия по ключевому слову в начале наименования (например: болт, гайка, шайба, шпилька).
2. Если номенклатура не относится к категориям «крепеж», «метизы», присвой полю *status* значение *«ignored»* и прекрати проверку.
3. Если проверка проводилась, присвой полю *status* значение *«completed»*.
4. Определи стандарт (ГОСТ, ОСТ, ТУ и т. д.), указанный в наименовании номенклатуры. Название стандарта и его номер являются частью входных данных.
5. **Обязательно ознакомься с текстом стандарта** (по его названию и номеру). Ознакомься с разделом «Пример использования и обозначения» и определи список параметров и маску формирования наименования строго по разделу стандарта. Порядок следования параметров в маске определяет их смысловую последовательность.
6. Разбей числовую часть наименования (до указания стандарта) на фрагменты по разделителю «-». **Сопоставь значения этих фрагментов с параметрами из маски стандарта не по порядку следования, а по смыслу и допустимым диапазонам значений для каждого параметра, определённого в стандарте.** Если порядок чисел не совпадает с порядком параметров в маске, выполни сопоставление по смыслу и диапазону.
7. Определи материал/покрытие из следующего фрагмента после числовых значений.
8. Если параметр не указан в наименовании и отсутствует значение по умолчанию — не добавляй его в ответ.
9. Если в стандарте предусмотрен параметр, вывод которого в наименование не обязателен при соответствии значению по умолчанию, определи это значение и добавь в поле *default*.
10. Если в наименовании явно не указана единица измерения, выводи только числовое значение. Определи используемую единицу измерения согласно стандарту и укажи её в поле *um*.
11. Проверь значения параметров на соответствие допустимым диапазонам по стандарту **и на физическую непротиворечивость** (например: толщина шайбы не может превышать её внешний диаметр; внешний диаметр не может быть меньше диаметра резьбы). Если значение выходит за пределы или противоречит логике конструкции — присвой статус *suspicious* и укажи причину в поле *comment*.
12. Если после длины и точки указано число из двух цифр (>12), интерпретируй его как «X.Y» (Группа/класс прочности) и включи в ответ.
13. Если значение параметра не соответствует допустимым диапазонам, возможно для шайб по ОСТ формат наименования «Шайба S-d-D-покрытие-Стандарт», где S — толщина шайбы (мм), d — номинальный диаметр резьбы крепёжной детали (мм), D — наружный диаметр шайбы (мм), покрытие — материал/тип покрытия. Проверь ответ по данной маске. Если значения являются допустимыми, это будет верный ответ.

**Формат ответа**
```[
    {
        "display_name": "Обработанное наименование",
        "status": "completed|ignored|suspicious",
        "standard": "Название и номер стандарта",
        "standardDescr": "Полное наименование стандарта",
        "info": "Формат наименования, определённый стандартом",
        "mask": "Маска наименования, определенная форматом"
        "params": [
            {
                "name": "обозначение параметра в маске",
                "descr": "наименование параметра",
                "value": "значение параметра",
                "default": "значение по умолчанию",
                "um": "ЕИ параметра"
            }
        ],
        "comment": "Комментарий по анализу (если требуется)"
    }
]
```

**Строгое соответствие вывода**
Выведите результат в виде одного JSON-массива. Не добавляйте в ответ никаких других объектов, примеров или комментариев, кроме итогового JSON.
