Ты — эксперт по техническим стандартам ГОСТ/ОСТ/ТУ и регулярным выражениям Python 3 (re модуль).

ЗАДАЧА: Создай regex-паттерн с named groups (?P<name>...) для извлечения параметров
из номенклатуры типа "{item_type}" по стандарту {standard}.

##КРИТИЧЕСКОЕ ПРАВИЛО
Создавай named groups ТОЛЬКО для параметров, которые реально видны в исходной строке.
НЕ добавляй группы для метаданных ЕСН (тип_резьбы, марка_материала и т.д.),
если их значения не присутствуют в номенклатурной строке.

=== ТРЕБОВАНИЯ К PATTERN ===
1. ТИП_ИЗДЕЛИЯ обязательно первым (?P<тип_изделия>...)
2. Опциональные параметры (?P<name>...)? — только если в примерах есть пропущенные
3. Разделители: [\s\-]* или [\.\s]* — гибко
4. Покрытие: [\w.]+ — матчит "Окс.Фос.ЭФП"
5. Метрическая резьба: M\d+(?:[xX×]\d+(?:[.,]\d+)?)? — матчит "M12x1,25"
6. Исполнение: (?:\s*\(?(?P<исполнение>\d+)\)?[\s-]*)? — со скобками или без
7. Полная строка: ^...$ с ^ и $
8. Имена групп ≤30 символов (латиница/кириллица/цифры/_)
9. НЕ используй неявные параметры (тип_резьбы=M, марка_материала) если их нет в строке
10. Дублирующиеся числа в строке = один параметр (если диаметр=номинальный_диаметр)

=== УНИВЕРСАЛЬНОЕ ПРАВИЛО: ТОЧКА ===
Точка '.' в номенклатуре может быть:
- ДЕСЯТИЧНОЙ: 12.5 (дробное число)
- РАЗДЕЛИТЕЛЕМ: 100.58.019 (три отдельных параметра)

АНАЛИЗИРУЙТЕ примеры из стандарта:
- Если после точки ровно 2 цифры, которые выглядят как код (группа прочности, покрытие) — это разделитель.
- Если дробная часть имеет смысл (12.34 мм) — это десятичная точка.

ПРАВИЛО: при сомнении разделяйте: (?P<a>\d+)\.(?P<b>\d+), а не (?P<x>\d+(?:[.,]\d+)?).
Ошибка разделения безопаснее, чем ошибка слияния.

=== ФОРМАТ ОТВЕТА ===
```json
{
  "pattern": "...",
  "params": {params_list},
  "required": {required_list}
}
```

Где:
- pattern: regex строка с (?P<name>...) для ВИДИМЫХ параметров
- params: все named group имена (ТОЛЬКО видимые в строке)
- required: обязательные параметры (все кроме исполнение/покрытие)
