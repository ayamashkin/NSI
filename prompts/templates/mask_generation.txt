ЗАДАЧА: Создай regex-паттерн с named groups (?P<name>...) для извлечения параметров из номенклатуры типа "{item_type}" по стандарту {standard}.

### КРИТИЧЕСКОЕ ПРАВИЛО
Создавай named groups ТОЛЬКО для параметров, которые реально видны в исходной строке.
НЕ добавляй группы для метаданных ЕСН (тип_резьбы, марка_материала и т.д.), если их значения не присутствуют в номенклатурной строке.
Примеры показывают "ВИДИМЫЕ В СТРОКЕ" и "МЕТАДАННЫЕ БД" — используй только ВИДИМЫЕ для regex.

=== ТРЕБОВАНИЯ К PATTERN ===

1. ТИП_ИЗДЕЛИЯ обязательно первым (?P<тип_изделия>...)
2. Опциональные параметры (?P<name>...)? — только если в примерах есть пропущенные
3. Разделители: [\s\-]* или [\.\s]* — гибко
4. Покрытие: [\w.]+ — матчит "Окс.Фос.ЭФП"
5. Метрическая резьба: M\d+(?:[xX×]\d+(?:[.,]\d+)?)? — матчит "M12x1,25"
6. Исполнение: (?:\s*\(?(?P<исполнение>\d+)\)?[\s-]*)? — со скобками или без
7. Полная строка: ^...$ с ^ и $
8. Имена групп ≤30 символов (латиница/кириллица/цифры/_)
9. НЕ используй неявные параметры (тип_резьбы=M, марка_материала) если их нет в строке
10. Дублирующиеся числа в строке = один параметр (если диаметр=номинальный_диаметр)

=== ФОРМАТ ОТВЕТА ===

```json
{
  "pattern": "...",
  "params": {params_list},
  "required": {required_list}
}
```

Где:
- pattern: regex строка с (?P<name>...) для ВИДИМЫХ параметров
- params: все named group имена (ТОЛЬКО видимые в строке)
- required: обязательные параметры (все кроме исполнение/покрытие)
