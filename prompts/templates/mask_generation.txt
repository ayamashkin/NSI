из номенклатуры типа "{item_type}" по стандарту {standard}.

##КРИТИЧЕСКОЕ ПРАВИЛО
Создавай named groups ТОЛЬКО для параметров, которые реально видны в исходной строке.
НЕ добавляй группы для метаданных ЕСН (тип_резьбы, марка_материала и т.д.),
если их значения не присутствуют в номенклатурной строке.

=== ТРЕБОВАНИЯ К PATTERN ===
1. ТИП_ИЗДЕЛИЯ обязательно первым (?P<тип_изделия>...)
2. Разделители МЕЖДУ параметрами: строго `[-\s]+` (тире или пробел/таб). НЕ используй `\s*` — оно не матчит тире!
   Пример правильно: `(?P<тип_изделия>Болт)[-\s]+(?P<длина>\d+)[-\s]+(?P<покрытие>[\w.]+)[-\s]+(?P<нтд_1>ОСТ\s*1\s*31141-80)`
   Пример НЕПРАВИЛЬНО: `(?P<тип_изделия>Болт)\s*(?P<длина>\d+)\s*(?P<покрытие>[\w.]+)\s*(?P<нтд_1>...)` — тире не матчит!
3. Покрытие: `[\w.]+` — матчит "Окс.Фос.ЭФП", "Кд.фос.окс", "Хим.Н"
4. Метрическая резьба: `M\d+(?:[xXхХ×]\d+(?:[.,]\d+)?)?` — матчит "M12x1,25", "M6х1"
5. Исполнение: `(?:\s*\(?(?P<исполнение>\d+)\)?\s*)?` — со скобками или без, опционально
6. Полная строка: `^...$` с `^` и `$`
7. Имена групп ≤30 символов (латиница/кириллица/цифры/_)
8. НЕ используй неявные параметры (тип_резьбы=M, марка_материала) если их нет в строке
9. Дублирующиеся числа в строке = один параметр (если диаметр=номинальный_диаметр)
10. `нтд_1` — ОБЯЗАТЕЛЬНЫЙ параметр, не опциональный. Запрещено `(?:...)?` вокруг стандарта.

=== УНИВЕРСАЛЬНОЕ ПРАВИЛО: ТОЧКА ===
Точка '.' в номенклатуре может быть:
- ДЕСЯТИЧНОЙ: 12.5 (дробное число)
- РАЗДЕЛИТЕЛЕМ: 100.58.019 (три отдельных параметра)
- ПОКРЫТИЯ: Кд3.фос.окс (точка — часть названия покрытия)

АНАЛИЗИРУЙТЕ примеры из стандарта:
- Если после точки ровно 2 цифры-кода (группа прочности, покрытие) — это разделитель.
- Если дробная часть имеет смысл (12.34 мм) — это десятичная точка.
- Если точка внутри названия покрытия (Кд.фос.окс) — это часть покрытия.

ПРАВИЛО: при сомнении разделяйте: `(?P<a>\d+)\.(?P<b>\d+)`, а не `(?P<x>\d+(?:[.,]\d+)?)`.
Ошибка разделения безопаснее, чем ошибка слияния.

=== ПРАВИЛО ДЛЯ ГОСТ ===
Для ГОСТ-стандартов (ГОСТ 7795-70, ГОСТ 7808-70 и т.д.):
- `класс_поле_допуска` — видимый параметр (6g, 8g). Пример: `M10-6gх45` → группа `(?P<класс_поле_допуска>\d+[a-z])`
- `группа_класс_прочности` — часто КОДИРУЕТСЯ числом после точки (56 = 5.6, 88 = 8.8). Если виден в строке — создавай группу.
- `свойства` — код (109, 88, 56). Если виден в строке — создавай группу.
- `марка_материала` — видна в строке (30ХГСА, 35Х). Создавай группу.
- `тип_резьбы` — M, Tr и т.д. Виден в строке. Создавай группу.

=== ФОРМАТ ОТВЕТА ===
```json
{
  "pattern": "...",
  "params": {params_list},
  "required": {required_list}
}
```

Где:
- pattern: regex строка с (?P<name>...) для ВИДИМЫХ параметров
- params: все named group имена (ТОЛЬКО видимые в строке)
- required: обязательные параметры (все кроме исполнение/шаг_резьбы/покрытие)
