Ты — эксперт по техническим стандартам ГОСТ/ОСТ/ТУ и регулярным выражениям Python.

ЗАДАЧА: Создай regex-паттерн с named groups (?P<name>...) для извлечения параметров из номенклатуры типа "{item_type}" по стандарту {standard}.

ВАЖНО: Ниже приведены реальные данные из Единого Справочника Номенклатуры (ЕСН).
Для КАЖДОЙ позиции показаны:
- Полное наименование (как в базе)
- ВСЕ заполненные поля ЕСН (материал, покрытие, диаметр, длина и т.д.)
- Разбор — где в строке находится каждый параметр

ИСПОЛЬЗУЙ эти данные чтобы понять СТРУКТУРУ строки и какие параметры где находятся.

=== ПРИМЕРЫ ИЗ ЕСН ({example_count} шт.) ===

{examples_text}

=== СТАТИСТИКА ЗАПОЛНЕННОСТИ ПОЛЕЙ (по {total_examples} примерам) ===
{stats_text}

ВАЖНО: В ответе перечисли ТОЛЬКО эти параметры: {params_hint}

ВАЖНО: В примерах выше есть варианты с пустыми/пропущенными параметрами (помечены "_variant": "без_..."). Если параметр отсутствует в наименовании — сделай его опциональным через `(?:...)?` в regex. Например исполнение в скобках — опционально.

{context_text}
=== ТРЕБОВАНИЯ К РЕГЕКСУ ===

1. Используй named groups (?P<param_name>...) — имя группы = имя поля из списка выше.
   Важно: используй ТОЛЬКО очищенные имена (без точек, скобок, запятых).
   Пример: поле "наружный_диаметр_(диаметр_вписанного_круга),_сторона_квадрата"
   -> в regex используй: (?P<наружный_диаметр_диаметр_вписанного_кру>...)
   ВАЖНО: имена групп ДОЛЖНЫ быть не длиннее 30 символов (обрезай если нужно).

2. Regex ДОЛЖЕН матчить ПОЛНУЮ строку от начала до конца (используй ^ и $)

3. Первое поле ВСЕГДА тип_изделия (НЕ наименование_типа!). Примеры показывают СТРУКТУРУ:
   [тип_изделия] [стандарт и разделители] [параметр1] [разделитель] [параметр2] ...
   Regex должен начинаться с: ^(?P<тип_изделия>...)\s+[стандарт]...

4. КРИТИЧЕСКОЕ ПРАВИЛО — проверка на дублирующиеся поля:
   Если два поля ЕСН имеют ОДИНАКОВОЕ значение в ОДНОЙ позиции строки — это ОДИН параметр в regex.
   Пример: если "наружный_диаметр=10" и "номинальный_диаметр_резьбы=10" оба = 10 в позиции "-10-",
   то в regex используй ТОЛЬКО один (?P<наружный_диаметр_...>10), а номинальный диаметр НЕ выделяй отдельно.
   Исключение: если в строке ЕСТЬ два разных числа (например "-10-12-"), тогда два параметра.

5. Для покрытия, материала и других текстовых полей используй `[\w.]+` вместо `\w+`
   (покрытия содержат точки: "Кд.фос.окс", "Окс.Фос.ЭФП", "Ц3.хр").
   Правильно: (?P<покрытие>[\w.]+)
   Неправильно: (?P<покрытие>\w+) — не матчит точки!

6. В конце строки могут быть лишние символы (запятая, точка с запятой и т.д.) — используй [\s,;.]*$ в конце regex

7. Поддерживай опциональные части через ?

8. Для разделителей используй [\.\s]* или [\s\-]*

9. Все числа ДОЛЖНЫ включать возможность десятичной точки/запятой

10. ОПЦИОНАЛЬНЫЕ ПАРАМЕТРЫ — КРИТИЧЕСКИ ВАЖНО:
    - Если в примерах ЕСН есть записи с ПРОПУЩЕННЫМ параметром — сделай его опциональным через `?`.
    - Некоторые параметры могут быть опциональными ДАЖЕ если все примеры содержат их.
      Например: исполнение в скобках (2), (3), (4) — это ОПЦИОНАЛЬНЫЙ параметр.
      В номенклатуре может быть как "Винт (2)-6-14-..." (со скобками), так и "Винт 6-14-..." (без скобок).
    - КРИТИЧЕСКОЕ ПРАВИЛО — разделитель ВНУТРИ опциональной группы:
      Неправильно: `(?:\s*\(\s*(?P<исполнение>\d+)\s*\)\s*)?(?P<диаметр>\d+)`
        — когда группа пропущена, после "Винт" сразу идёт диаметр (OK).
        — но со скобками: после "(2)" нет разделителя "-", диаметр не сматчит.
      Правильно:  `(?:\(\s*(?P<исполнение>\d+)\s*\)\s*-\s*)?(?P<диаметр>\d+)`
        — разделитель "(2)-" целиком внутри опциональной группы.
        — без скобок: группа пропущена, диаметр сматчит сразу.
        — со скобками: группа match "(2)-", диаметр сматчит дальше.
    - КОПИРУЙ этот шаблон (замени только название стандарта и порядок параметров):
      `^Винт(?:\(\s*(?P<исполнение>\d+)\s*\)\s*-\s*)?(?P<номинальный_диаметр_резьбы>\d+)\s*-(?P<длина>\d+)\s*-(?P<покрытие>[\w.]+)\s*-ОСТ\s*1\s*31503-80[\s,;.]*$`
      Параметры:
        исполнение = целое число в скобках: \d+  (НЕ \d+(?:\.\d+)? — исполнение всегда целое!)
        номинальный_диаметр_резьбы = \d+
        длина = \d+
        покрытие = [\w.]+  (НЕ \w*\.?\w* — покрытие может содержать НЕСКОЛЬКО точек!)
      Этот regex матчит ВСЕ 4 варианта:
        Винт (2)-10-16-Хим.Пас-ОСТ 1 31503-80
        Винт 10-16-Хим.Пас-ОСТ 1 31503-80
        Винт (4)-5-10-Хим.Пас-ОСТ 1 31503-80
        Винт 5-10-Хим.Пас-ОСТ 1 31503-80
